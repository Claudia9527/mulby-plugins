> 版本：v2.0 Draft

## 1. 概述

### 1.1 背景

「待办番茄」是 Mulby 平台上的轻量待办管理插件，已实现待办 CRUD、快速捕获、置顶便签、番茄钟和 AI 拆解/总结等核心功能。v1.0 上线后，在实际使用中暴露出以下核心短板：

- 待办缺乏时间维度和优先级维度，无法帮助用户排序决策
- 数据操作不可逆，误删无法恢复
- 便签窗和番茄钟的交互能力较弱
- 专注数据只有"今日"粒度，无法回顾历史趋势
- AI 能力仅覆盖"拆解"和"总结"两个场景，价值密度不够

### 1.2 目标

v2.0 优化方向聚焦于 **可用性提升** 和 **数据价值挖掘**，具体目标：

| 指标 | 目标 |
|------|------|
| 用户日活使用时长 | 提升 30%（通过番茄钟沉浸感和便签窗实用性） |
| 待办完成率 | 提升 20%（通过优先级引导和 AI 日规划） |
| 用户留存 | 减少因误删导致的弃用 |

### 1.3 优先级定义

| 等级 | 含义 | 时间窗口 |
|------|------|---------|
| P0 | 核心体验缺陷，必须修复 | v2.0 首批 |
| P1 | 高价值特性，显著提升竞争力 | v2.0 第二批 |
| P2 | 长期迭代方向 | v2.x |

## 2. P0 需求：核心体验修复

### 2.1 截止日期与优先级

#### 2.1.1 需求描述

用户需要明确知道"哪个任务最紧急、什么时候截止"。当前数据模型缺乏时间和优先级维度，导致待办列表沦为无差别的流水账。

#### 2.1.2 数据模型变更

在 `TodoItem` 类型中新增以下字段：

```typescript
interface TodoItem {
  // ... 现有字段保持不变
  dueDate?: number       // 截止时间戳（精确到天）
  priority?: 'high' | 'medium' | 'low'  // 优先级，默认不设置
}
```

#### 2.1.3 功能规格

**创建时设置：**
- 新建待办输入框右侧新增两个可选按钮：📅 截止日期（弹出日期选择器）、🔴🟡🟢 优先级（下拉选择）
- 不设置时字段为 `undefined`，不影响现有逻辑

**列表排序逻辑（新默认排序）：**
1. 置顶 (`pinned`) 最优先
2. 已完成排最后
3. 未完成项内部按以下规则排序：
   - 过期未完成 > 今日截止 > 明日截止 > 有截止日期 > 无截止日期
   - 同截止日期内：high > medium > low > 无优先级
   - 同优先级内：按更新时间倒序

**视觉标记：**
- 过期任务：标题左侧显示红色警告图标，截止日期文字标红
- 今日截止：橙色标记
- 高优先级：标题前显示 🔴 圆点
- 中优先级：标题前显示 🟡 圆点
- 低优先级：标题前显示 🟢 圆点

**快速捕获语法扩展：**

| 语法 | 含义 | 示例 |
|------|------|------|
| `todo !标题` | 高优先级 | `todo !紧急修复线上Bug` |
| `todo !!标题` | 中优先级 | `todo !!完善文档` |
| `todo !!!标题` | 低优先级 | `todo !!!整理书签` |
| `todo 标题 @明天` | 截止日期为明天 | `todo 提交周报 @明天` |
| `todo 标题 @下周一` | 截止日期为下周一 | `todo 准备PPT @下周一` |
| `todo 标题 @6.15` | 截止日期为指定日 | `todo 发布v2 @6.15` |

解析规则：优先级标记 `!` 在标题最前面，日期标记 `@` 在标题最后面。两者可组合使用。

#### 2.1.4 便签窗联动

便签窗中的待办项也需要展示截止日期和优先级标记（圆点 + 日期文字）。

#### 2.1.5 兼容性

- 旧数据中不存在 `dueDate` 和 `priority` 字段，读取时视为 `undefined`
- `normalizeState()` 函数自动兼容，无需数据迁移

### 2.2 删除撤销机制

#### 2.2.1 需求描述

当前删除是不可逆操作。在键盘快捷键模式下，用户容易误触 `d` 键导致数据丢失，损害使用信任感。

#### 2.2.2 方案：Toast 撤销（推荐）

采用轻量级 Toast 撤销方案，不引入回收站的复杂性：

**交互流程：**
1. 用户按 `d`（或 `Shift+D` 直接删除、点击删除按钮）
2. 待办项立即从列表中移除（乐观更新）
3. 底部弹出 Toast 通知：`"已删除「{标题}」" [撤销]`
4. Toast 持续 5 秒，倒计时展示
5. 用户点击「撤销」→ 待办项恢复到原位置
6. 5 秒后 Toast 消失 → 执行真正的 `removeFromStorage` 操作
7. 快捷键 `Ctrl+Z` / `Cmd+Z` 在 Toast 可见期间也触发撤销

**实现要点：**
- 维护一个 `pendingDeletion: {item: TodoItem, timeoutId: number} | null` 状态
- 列表渲染时过滤掉 `pendingDeletion.item`
- 撤销时清除 timeout 并将 item 重新插入列表
- 如果在前一个待删除项未确认时又删除了新项：前一个立即执行删除，新项进入待删除状态
- 不修改 `d` 键的二次确认逻辑，保留作为第一道防线

#### 2.2.3 键盘快捷键

| 操作 | 说明 |
|------|------|
| `d` | 第一次按标记为待删除（提示确认），第二次按进入 Toast 撤销流程 |
| `Shift+D` | 直接进入 Toast 撤销流程（跳过二次确认） |
| `Ctrl+Z` / `Cmd+Z` | Toast 可见期间执行撤销 |

### 2.3 便签窗增强

#### 2.3.1 需求描述

便签窗目前的定位是"只读缩略视图"，功能过于单薄：最多显示 8 条、不能添加、不能编辑。作为一个独立窗口，应具备轻量但完整的操作能力。

#### 2.3.2 功能规格

**新增快速添加：**
- 便签窗顶部新增一个紧凑输入框，`placeholder="快速添加…"`
- Enter 提交，Esc 清空
- 保持便签窗的视觉风格（暖色调便签纸感）

**取消 8 条硬编码限制：**
- 移除 `MAX_VISIBLE = 8` 限制
- 改为滚动展示所有未完成待办
- 底部不再显示"还有 N 项…"，改为实际滚动

**支持拖拽排序：**
- 长按（或鼠标拖拽）可调整待办顺序
- 排序结果持久化到 `TodoItem` 的 `sortOrder` 字段

**展示优先级和截止日期：**（依赖 2.1）
- 每条待办项右侧紧凑显示截止日期（如"明天"、"6/15"）
- 优先级用颜色圆点标记

#### 2.3.3 窗口尺寸

- 默认尺寸从 320×420 调整为 320×480（增加高度以容纳输入框）
- 最小尺寸：280×300

## 3. P1 需求：高价值特性

### 3.1 番茄钟历史数据与可视化

#### 3.1.1 需求描述

当前统计仅记录 `pomodoroToday` 和 `pomodoroTotal`，每日数据在跨日时被清零。用户无法看到趋势，缺乏成就感反馈。

#### 3.1.2 数据模型变更

```typescript
interface DailyRecord {
  date: string          // 'YYYY-MM-DD'
  pomodoroCount: number
  focusMinutes: number
  completedTodos: number
}

interface Stats {
  // ... 保留现有字段
  dailyHistory: DailyRecord[]  // 最近 90 天的每日记录
}
```

#### 3.1.3 功能规格

**数据记录：**
- 每完成一个番茄钟，自动追加到当天的 `DailyRecord`
- 每完成一个待办，`completedTodos` 自增
- 超过 90 天的记录自动清理（控制存储大小）

**可视化（主面板新增"统计"Tab）：**
- 默认展示最近 7 天的柱状图（每天的番茄数和专注分钟数）
- 可切换为 30 天视图
- 显示本周/本月汇总：总番茄数、总专注时长、日均数据
- 每个待办项展示累计专注时间徽章（当前 `focusMinutes` 字段已存在，仅需 UI 展示）

**番茄钟界面改进：**
- 完成一个番茄钟后显示简洁的完成动画（圆环变绿 + 对勾）
- 当前任务名称下方显示"已专注 N 个番茄"

#### 3.1.4 筛选 Tab 扩展

主面板的 Tab 从 3 个扩展为 4 个：

| 序号 | Tab | 快捷键 |
|------|-----|--------|
| 1 | 全部 | `1` |
| 2 | 进行中 | `2` |
| 3 | 已完成 | `3` |
| 4 | 统计 | `4` |

### 3.2 子任务/清单

#### 3.2.1 需求描述

复杂任务需要分解步骤。AI 拆解虽然能生成多条待办，但它们之间没有层级关系，全部平铺在列表中会显得混乱。

#### 3.2.2 数据模型变更

```typescript
interface ChecklistItem {
  id: string
  text: string
  done: boolean
}

interface TodoItem {
  // ... 现有字段
  checklist?: ChecklistItem[]  // 子任务清单
}
```

选择 `checklist` 而非 `parentId` 树形结构的原因：
- 复杂度低，不需要递归渲染
- 更符合待办清单的使用直觉（勾选子项）
- 避免无限嵌套带来的 UI 复杂性

#### 3.2.3 功能规格

**创建子任务：**
- 展开某个待办项时，下方出现子任务列表 + 添加输入框
- Enter 添加子任务，最多 20 个

**进度显示：**
- 父待办项右侧显示进度 badge：`3/5`（已完成/总数）
- 所有子任务完成时，父待办项自动标记为完成（可配置）

**AI 拆解联动：**
- AI 拆解结果新增选项"作为子任务导入到…"
- 选择一个已有待办，拆解结果写入其 `checklist`

**键盘快捷键：**
- `Tab` 展开/折叠选中项的子任务
- 在子任务模式下，`n` 添加新子任务，`j/k` 在子任务间移动

---

### 3.3 AI 能力增强

#### 3.3.1 智能优先级建议

用户添加待办后，AI 根据标题语义自动推荐优先级：
- "紧急/尽快/今天必须" → 建议高优先级
- 含日期信息 → 自动提取截止日期
- 提示弹窗让用户确认或忽略

#### 3.3.2 每日规划

主面板新增"今日规划"入口：
- AI 综合未完成待办的截止日期、优先级、历史专注数据
- 生成建议的今日执行顺序（3-5 条）
- 用户可一键设为"今日焦点"（pinned）

#### 3.3.3 每日复盘

每天首次打开时自动触发（可关闭）：
- 对比昨日待办完成情况
- 未完成项自动提醒
- 给出效率趋势简评

## 4. P2 需求：长期迭代

### 4.1 重复/周期任务

```typescript
interface TodoItem {
  // ...
  recurrence?: {
    type: 'daily' | 'weekly' | 'monthly'
    interval: number  // 每 N 天/周/月
    nextDue?: number  // 下次触发时间
  }
}
```

完成一个周期任务后，自动创建下一期（标题相同、截止日期顺延）。

### 4.2 数据导出/备份

- 设置页新增"导出数据"按钮，支持 JSON 和 CSV 格式
- 支持"导入数据"，合并或覆盖现有待办
- 导出内容包含：待办列表、设置、历史统计

### 4.3 番茄钟沉浸增强

- 自定义提示音（内置 3-5 种 + 自定义上传）
- 白噪音背景（雨声、咖啡厅、森林）
- 休息活动建议卡片（站起来、喝水、做拉伸）
- 可选"专注锁定"模式：番茄钟运行时隐藏待办列表，仅显示计时器

### 4.4 多端数据同步

若 Mulby 平台支持云存储能力，可考虑：
- 待办数据云端同步
- 多设备间实时更新
- 冲突解决策略（最后写入优先 + 手动合并）
